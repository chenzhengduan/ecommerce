# 阿里巴巴电商知识库刷新运行手册

> 用途：后续每次刷新阿里巴巴电商知识库时，按本文档执行。

## 固定流程

1. 生成知识清单。

   ```bash
   python3 tools/knowledge_refresh.py --docs-dir alibaba-ecommerce --out-dir alibaba-ecommerce/00-知识刷新/inventory
   ```

2. 对照 [主题地图](主题地图.md)，找出缺失主题。

3. 标记过时风险主题。

   优先检查平台规则、费用、信保、RFQ、广告、物流、支付、争议、认证和合规内容。

4. 选择本轮范围。

   每次默认处理：

   - 1 到 3 个缺失主题。
   - 1 到 3 个过时或不完整主题。

5. 查验来源。

   - 稳定概念可基于本地知识整理。
   - 平台规则、费用、交易保障、支付、物流和合规必须查验当前来源。
   - 无法确认的内容记录为待跟进，不写成确定结论。

6. 更新内容并生成报告。

7. 验证网页。

   ```bash
   .venv/bin/mkdocs build --strict
   ```

   然后确认本地网页能访问新增或更新页面。

## 完成标准

- 刷新报告已生成。
- 新增和更新文件已列出。
- 时效性事实有来源依据或待跟进说明。
- `mkdocs build --strict` 通过。
- 本地网页能访问阿里巴巴知识库页面。
