## 学习内容摘要

### 1. 广告归因模型基础

**什么是归因模型？**
- 归因模型定义了如何将销售功劳分配给营销漏斗中不同的触点
- 帮助卖家理解哪些广告活动、关键词或投放策略实际带来了转化

**亚马逊主要归因模型**
- **首次触点归因（First-Touch）**：100%功劳归给用户首次接触的广告
  - 适用于 awareness 阶段，帮你找到哪些渠道能带来新客户
- **末次触点归因（Last-Touch）**：100%功劳归给最终转化前的最后一次触点
  - 最常用，但会低估 awareness 和 consideration 阶段的工作
- **线性归因（Linear）**：平均分配功劳给所有触点
  - 适合客户决策路径均匀分布的场景
- **时间衰减归因（Time Decay）**：近期触点获得更多功劳
  - 适合决策周期短的产品
- **位置归因（Position-Based）**：首尾各40%，中间均分20%
  - 兼顾拉新和转化

### 2. 亚马逊归因报告解读

**Advertising Report 关键指标**
- **Attributed Sales**：归因销售额，该广告活动带来的销售额
- **Attributed Orders**：归因订单数
- **Attributed Conversion Rate**：归因转化率
- **ROAS**：广告支出回报率 = 归因销售额 / 广告支出

**归因窗口**
- 亚马逊默认归因窗口：点击后7天内（适用于大多数品类）
- 可以调整：1天、7天、14天、30天
- 归因窗口长 = 归因销售额高，但可能包含非广告触发的订单

### 3. 多触点分析实战

**客户旅程拆解**
1. **Awareness（认知）**：用户首次看到你的广告
   - 广告类型：Sponsored Display、 Sponsored Videos
   - 指标：Impressions、Clicks、Reach
2. **Consideration（考虑）**：用户开始研究和比较
   - 广告类型：Sponsored Products、Sponsored Brands
   - 指标：Detail Page Views、Add to Cart
3. **Conversion（转化）**：用户完成购买
   - 广告类型：Sponsored Products（购买）
   - 指标：Orders、Sales

**多触点分析的价值**
- 帮你识别不同阶段的表现，平衡预算分配
- 避免仅看 ROAS 导致的 "last-touch bias"（末触偏见）
- 发现跨渠道协同效应：Display 带来的流量，最终被 Sponsored Products 转化

### 4. 归因数据优化策略

**基于归因调整预算**
- 如果 First-Touch 触发的广告 ROAS 低但带来了大量新客，考虑增加认知阶段预算
- 如果某渠道的归因转化率高，但整体份额低，说明它扮演了 "closer" 角色

**常见问题与解决**
- 归因数据与实际销售不符：检查归因窗口，排除自然搜索干扰
- 多渠道归因复杂：使用亚马逊 Attribution（归因）工具整合站外流量

### 5. 站外归因追踪

**amazon Attribution**
- 用于追踪站外流量（Facebook、Google、TikTok、Email）带来的亚马逊站内转化
- 创建追踪链接，生成唯一 URL
- 跟踪指标：Click-through、Add to Cart、Purchase

**归因流程**
1. 创建 Attribution 项目和追踪链接
2. 在站外渠道使用追踪链接
3. 查看亚马逊后台 Attribution 报告
4. 分析不同站外渠道的归因销售额
