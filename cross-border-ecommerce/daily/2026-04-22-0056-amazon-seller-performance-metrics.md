## 学习内容摘要

### 核心绩效指标体系

亚马逊卖家绩效（Performance）直接决定账户生死线。亚马逊使用**账户健康评级（Account Health Rating）**系统，以颜色+分数形式呈现：

- **绿色（优秀）：** 无违规或轻微违规
- **黄色（警告）：** 多项或重复轻微违规
- **红色（危险）：** 严重违规，账户面临暂停风险

### 六大核心指标

| 指标 | 全称 | 健康阈值 | 说明 |
|------|------|---------|------|
| **CR** | 完美订单率 (Perfect Order Rate) | >98% | 包含：无差评、无A-Z、无退款、无取消 |
| **ODR** | 订单缺陷率 (Order Defect Rate) | <1% | = (差评+Refunded+A-Z) / 总订单数 |
| **LSR** | 迟发率 (Late Shipment Rate) | <4% | 在预计发货日期后填写的订单比例 |
| **VTR** | 有效追踪率 (Valid Tracking Rate) | >95%（美国站） | 所有可追踪的包裹需提供有效追踪号 |
| **FR** | 发票缺陷率 (FBA缺陷率通常为0) | <1% | 预发索要率（FBM相关） |
| **RTD** | 按时交货率 (On-Time Delivery) | 参考VTR | 实际送达日期 vs 承诺日期 |

### 绩效计算周期

- **订单缺陷率（ODR）：** 过去60天或90天滚动计算
- **迟发率（LSR）：** 过去7天和30天滚动
- **有效追踪率（VTR）：** 过去30天
- **FBA卖家的天然优势：** FBA订单不计入LSR（亚马逊负责配送），但VTR由亚马逊负责

### 绩效改善策略

**降低ODR：**
1. 主动联系差评客户，解决问题
2. 使用"联系买家"功能请求移除负面反馈
3. Product Returns → 分析高频退款原因，优化产品/页面
4. A-to-Z Claim → 及时响应，90%主动解决可避免claim成立

**降低LSR：**
1. FBA转移：自发货转FBA可彻底解决LSR问题
2. 备货提前量：确保库存充足，避免断货
3. 设置缓冲时间：预计发货时间（handling time）留出余量
4. 使用"延迟发货"功能：如有不可抗力，主动延迟而非逾期

**提升VTR：**
1. 使用与亚马逊打通的物流商（UPS、FedEx、DHL、USPS）
2. 发货当日确认追踪号
3. 取消率控制：及时取消无效订单
4. 高质量包装减少丢失率

### 账户健康预警系统

- **绩效通知（Performance Notifications）：** 出现在卖家中心后台，首页右上角
- **Account Health页面：** 完整展示所有指标状态
- **Actionable Alerts：** 亚马逊会提前预警，给出改善建议
- **Account suspended风险：** 三类高危行为直接触发：严重知识产权投诉、ODR>1%、伪造追踪号

## 学习内容评估

| 评估维度 | 判断 |
|---------|------|
| **是否正确** | ✅ 正确。亚马逊绩效指标体系是公开知识，阈值和计算方式与实际情况吻合。ODR<1%、LSR<4%、VTR>95%是官方标准。 |
| **是否新知识** | ⚠️ 部分新知。ODR/LSR/VTR基础概念早已覆盖，但**CR（完美订单率）作为独立指标**、**RTD（按时交货率）**、以及**账户健康仪表盘的集成视图**，之前没有系统梳理过。 |
| **是否好理解** | ✅ 好理解。绩效指标本质是"百分比=问题单/总单"，逻辑清晰。但需要注意：FBA和FBM的LSR计算逻辑不同，VTR在美国站和欧洲站要求不同，容易混淆。 |

## 补充思考

**CR vs ODR：** 两者高度相关但不完全重叠。CR是更严格的标准（要求完美），ODR是亚马逊官方绩效指标。CR>98%通常意味着ODR<1%，但CR可以作为更精细的健康度参照。实际运营中，**优先控制ODR<1%，CR作为辅助观察指标**。

**FBA vs FBM绩效差异：** FBA卖家可以彻底规避LSR问题（亚马逊负责配送），但这不意味着FBA账号免疫——ODR和VTR仍然是FBA账户的核心风险点。FBM卖家则需要同时关注LSR、VTR、迟发率，对运营精细度要求更高。

**绩效申诉的核心逻辑：** 当绩效指标超标时，申诉POA需要包含：问题根因分析（Root Cause）+ 已采取的纠正措施（Corrective Actions）+ 预防机制（Preventive Measures）。亚马逊不仅要看结果改善，更要看"你是否有系统性防止问题复发的机制"。
