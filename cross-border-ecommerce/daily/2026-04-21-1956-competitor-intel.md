## 一、核心概念

### 什么是竞品监控（Competitive Intelligence）？
通过系统化收集、分析竞争对手在亚马逊平台上的关键数据，为选品、定价、广告和运营决策提供数据支撑。

### 竞品反查（Reverse ASIN Lookup）
通过已知的ASIN，反向查询其流量词、广告词、销量、评论等数据，是做竞品分析和选品调研的核心手段。

## 二、核心竞品数据维度

### 2.1 基础数据
| 数据项 | 说明 | 获取工具 |
|--------|------|---------|
| BSR排名 | 畅销排名，反映销量 | Keepa, Jungle Scout |
| 评论数量 | 累计评价总数 | 页面直接查看 |
| 评分星级 | 4.3+为安全线 | 页面直接查看 |
| 上架时间 | 判断竞争时长 | Keepa历史记录 |
| FBA/FBM | 判断物流模式 | 页面信息 |
| 变体数量 | 子ASIN数量 | 页面直接查看 |

### 2.2 销售数据
| 数据项 | 说明 |
|--------|------|
| 月销量估算 | 根据BSR反推 |
| 日均销量 | 月销量/30 |
| 销售额估算 | 月销量 × 单价 |
| 库存深度 | 是否有FBA库存预警迹象 |

### 2.3 关键词数据
| 数据项 | 说明 |
|--------|------|
| 流量词（Traffic Terms） | 带来自然流量的核心关键词 |
| 广告词（Ad Terms） | 竞争对手投放广告的关键词 |
| 关键词排名 | 某词下的BSR排名 |
| 搜索量 | 月搜索量（反映市场需求） |

### 2.4 广告数据（最难获取）
| 数据项 | 说明 |
|--------|------|
| 广告类型 | SP / SB / SD |
| 广告位置 | 搜索结果顶部/产品页/其余 |
| 广告竞价 | 估算值（工具推算） |
| 广告文案 | 吸引点击的写法 |

## 三、主流竞品分析工具

### 3.1 Helium 10（综合性平台）
- **Cerebro**：反查竞品ASIN的流量词，支持查看词下的竞品数量和排名分布
- **Black Box**：选品神器，支持按20+个维度筛选产品
- **Frankenstein**：将关键词列表处理成分词，用于优化Search Terms
- **Market Tracker 360**：监控竞品价格、评分、BSR变化
- **益处**：工具链完整，数据覆盖面广
- **不足**：价格较贵（$99/月起）

### 3.2 Jungle Scout
- **Product Database**：数据库选品
- **Competitor Intelligence**：竞品数据分析
- **Keyword Scout**：关键词反查
- **历史数据**：最长可看3年BSR历史
- **益处**：数据准确度高，适合大规模调研
- **不足**：部分功能需要高配套餐

### 3.3 Keepa（价格/排名追踪）
- **历史BSR**：可视化长期排名趋势
- **价格历史**：追踪价格变动
- **评论增长**：追踪评价数量变化
- **Prime Pantry标识**：识别特殊物流
- **益处**：浏览器插件免费，数据直观
- **不足**：不支持关键词反查

### 3.4 Helium 10 vs Jungle Scout对比
| 功能 | Helium 10 | Jungle Scout |

| 反查关键词 | ✅ Cerebro强 | ✅ Keyword Scout |

| 广告数据 | 部分 | 部分 |

## 四、关键词反查实操（Cerebro为例）

### 第一步：输入竞品ASIN
- 输入类目BSR前10的ASIN
- 同时反查3-5个竞品，识别共同高流量词

### 第二步：分析关键词数据
关注指标：
- **搜索量（Search Volume）**：越大说明需求越高
- **竞品数量（# of Products）**：该词下有多少产品
- **排名集中度（Rank Distribution）**：头部产品占比
- **PPC竞价区间（PPC Bid Range）**：广告成本参考

### 第三步：筛选关键词
分层逻辑：
| 层级 | 特征 | 用途 |
|------|------|------|
| 大词（Head Terms） | 搜索量>10万，竞品>1000 | 品牌广告/防守 |
| 中等词（Mid Terms） | 搜索量1-10万，竞品300-1000 | 自然排名主攻 |
| 长尾词（Long Tail） | 搜索量<1万，竞品<300 | 新品切入首选 |

### 第四步：验证关键词真实效果
- 用Helium 10的"SERP Watch"功能监控排名变化
- 记录优化前后的自然排名提升情况

## 五、竞品监控实战流程

### 5.1 日常监控（每周一次）
```
✅ 检查竞品BSR排名变化（是否被反超）
✅ 检查竞品评分变化（新出差评？）
✅ 检查竞品价格变化（是否在降价促销）
✅ 检查竞品变体变化（是否新增了规格）
✅ 检查竞品评论增长（增速如何）
```

### 5.2 竞品突然发力时的应对
当发现竞品BSR急剧上升：
1. 立即用Cerebro反查对方近期投放的关键词
2. 查看其是否在大幅降价
3. 查看其是否有新的评价涌入（Vine？）
4. 评估是否需要调整广告竞价或开启竞品防御广告

### 5.3 月度竞品报告模板
```
竞品名称：[竞品ASIN/品牌]
监控周期：[月份]
数据摘要：
- 平均BSR：___
- 月均评论增量：___
- 平均价格：___
- 主关键词排名变化：___

主要发现：
1. [竞品最近在___词上排名提升，疑似在投放广告]
2. [竞品价格下调了___%，可能影响Buy Box]
3. [竞品新增了___变体，正在拓展规格]

应对策略：
1. [针对___词提升bid]
2. [调整定价至___]
3. [___]
```

## 六、关键词反查在选品中的应用

### 6.1 蓝海词挖掘
通过Cerebro找出同时满足：
- 月搜索量 > 5000
- 竞品数量 < 50
- 该词下BSR前10中有新品（上架<6个月）

→ 说明该词竞争不充分，有新品空间

### 6.2 判断市场容量
- 某词下所有竞品月销量之和 = 该词市场容量
- 计算方式：在词下取BSR第1/5/10名分别反推销量，加总估算

### 6.3 寻找竞争对手弱点
- 差评中提炼共性痛点 → 自己的产品差异化方向
- 高分差评（4星）的"but..."部分 → 改进空间

## 七、店铺级竞品分析

### 7.1 分析维度
| 维度 | 关注点 |
|------|--------|
| 产品线宽度 | 对方有多少SKU |
| 价格带分布 | 对方覆盖哪些价格区间 |
| 爆款集中度 | 1-2个爆款还是分散 |
| 新品比例 | 近期上架了多少新品 |
| 评论分布 | 是否集中在爆款 |

### 7.2 识别竞争对手的运营策略
| 特征 | 策略判断 |
|------|---------|
| 高评分（4.7+）大量评论 | 重评价运营，S单+Vine |
| 价格明显低于市场 | 成本优势 or  清仓 |
| 大量变体 | 测款能力强，SKU矩阵 |
| 高广告曝光 | 付费流量依赖型 |
| 多平台同款 | 供应链强，泛平台运营 |

## 八、我的评价

### 是否正确？
✅ **核心逻辑正确**：关键词反查 → 流量词分析 → 决策支持，这条链路是行业通用做法。

⚠️ **部分数据精度**：BSR反推销量是估算，有误差；各工具的搜索量数据口径不完全一致。

⚠️ **广告数据是盲区**：亚马逊不公开竞品广告数据，所有工具的PPC竞价都是估算，不是真实值。

### 是否是新知识？
**部分新**：对于已学过Jungle Scout/Helium 10基础功能的同学，关键词反查是已知工具的具体操作细节。但如果之前没有系统了解Cerebro的实操流程，这一节属于新知识。

**补充理解**：
- 关键词反查（流量词反查）是Helium 10 Cerebro的核心功能，Jungle Scout对应工具是Keyword Scout
- 竞品监控和选品是紧密联系的，不是两个独立流程

### 是否好理解？
**✅ 理解度较高**：
- 工具功能描述清晰，对应关系明确
- 关键词分层（大词/中词/长尾）是运营常识，理解门槛低
- 实操流程（4步法）容易转化为行动

**需要补充的点**：
- 实际操作中建议先用Helium 10免费版（Chrome插件）试一下Cerebro功能，看数据颗粒度是否满足需求
- 没有工具的情况下，Keepa插件看历史BSR曲线也能判断竞品大致走势

## 九、关键要点速记

> 竞品监控核心 = BSR追踪 + 价格追踪 + 评论追踪 + 关键词反查
>
> 关键词反查三步：找词 → 筛词 → 验证效果
>
> 蓝海词特征：搜索量高 + 竞品少 + 有新品空间
>
> 监控频率：日常（每日看）→ 周报 → 月度报告
>
> 广告数据是竞品分析最大盲区，所有PPC数据都是估算值
