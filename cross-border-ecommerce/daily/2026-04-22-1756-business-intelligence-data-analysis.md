## 一、学习内容摘要

### 1. 商业智能（BI）在跨境电商中的核心价值

跨境电商BI是将分散在亚马逊、Shopify、Facebook Ads、Google Ads、物流商、ERP等平台的数据进行采集、清洗、分析、可视化，最终支持运营决策的系统工程。

**核心目标：**
- 数据驱动决策替代"凭经验拍脑袋"
- 打通数据孤岛，实现全链路可视化
- 发现业务问题于萌芽阶段
- 预测趋势，提前布局

### 2. 跨境电商典型数据源与整合

| 数据源 | 数据类型 | 获取方式 |
|--------|----------|----------|
| 亚马逊 Seller Central | 订单、库存、广告、财务、绩效 | SP API、Selling Partner API |
| Shopify 独立站 | 流量、订单、弃购、客户 | Shopify API、Storefront API |
| Facebook/Meta Ads | 广告投放数据、受众数据 | Meta Business API |
| Google Ads | 搜索广告、Shopping广告数据 | Google Ads API |
| 物流渠道 | 物流轨迹、运费成本 | 物流商API、CSV导出 |
| 财务系统 | 收款、支出、利润 | ERP系统、PayPal/Stripe报表 |
| 客服系统 | 差评、响应时长、退换货 | Zendesk/Gorgias API |

**数据整合的三种模式：**
1. **手动导出合并**（Excel/Google Sheets）—— 适合初创期
2. **API + 中间件**（Zapier/Make + Google Sheets）—— 适合成长期
3. **数据仓库 + BI工具**（BigQuery/PostgreSQL + Looker Studio）—— 适合规模化

### 3. 亚马逊SP API数据分析实战

SP API（ Selling Partner API）是亚马逊官方数据接口，覆盖以下核心模块：

- **Orders API**：订单详情、退货、退款数据
- **Finances API**：收款、费用、退款明细
- **Sales API**：销售量、销售额、热销榜
- **Inventory API**：库存水平、可售天数、库龄
- **Advertising API**：SP/SB/SD广告数据、竞价、出价
- **Recommendations API**：亚马逊智能建议

**常用分析维度：**
- 销售看板：销售额、订单量、退货率、客单价
- 广告看板：ACOS、TACOS、RoAS、关键词表现
- 库存看板：DoS（Days of Supply）、IPI、库龄分布
- 财务看板：毛利率、各项费用占比、现金流

### 4. Shopify BI分析核心指标

| 维度 | 核心指标 |
|------|----------|
| 流量 | 会话量、跳出率、页面/会话 |
| 转化 | 转化率、加购率、弃购率 |
| 订单 | 订单数、客单价、GMV |
| 客户 | 新客户率、LTV、CAC、复购率 |
| 财务 | 毛利率、廣告支出占比、广告回收周期 |

**Shopify Analytics + GA4 联动：**
- Shopify Analytics 负责电商核心数据
- Google Analytics 4 负责流量和行为数据
- 通过 Client ID 关联实现跨平台用户路径分析

### 5. BI工具生态与选型

**国际化 BI 工具：**
- **Looker Studio**（原 Data Studio）：免费、模板丰富、Google生态整合，适合中小卖家
- **Power BI**：微软生态、与Excel高度整合，适合需要本地化部署的企业
- **Tableau**：可视化能力强，适合深度数据分析，但成本较高

**垂直电商 BI 工具：**
- **DataHawk**：亚马逊+独立站一站式分析，SP API直连
- **Helium 10**：选品+关键词+广告分析为主
- **Sellics**：亚马逊广告+财务+库存综合分析
- **Perpetua**：广告自动化+数据洞察

**自建 BI 架构（适合中大型卖家）：**
```
数据源（API/CSV）→ 数据管道（Fivetran/Airbyte）→ 数据仓库（BigQuery/Snowflake/PostgreSQL）→ BI层（Looker Studio/Metabase）→ 决策输出
```

### 6. 数据驱动决策框架（PDCA + RICE）

**日常运营决策：**
- 数据采集 → 异常检测 → 假设生成 → 测试验证 → 固化流程

**爆款分析决策：**
- 销量突增 → 归因分析（广告？评价？竞品？）→ 快速复制
- 评价突增 → 分析来源渠道 → 评估质量

**财务决策：**
- 毛利率监控 → 成本异常预警 → 供应商谈判依据
- ACOS突增 → 广告结构审查 → 竞价策略调整

### 7. 数据中台建设三阶段

**第一阶段（0-3个月）：数据整合**
- 打通亚马逊SP API + Shopify API + 广告平台
- 建立每日数据同步机制
- 构建基础运营看板

**第二阶段（3-6个月）：深度分析**
- 建立产品线profit & loss（P&L）报表
- 客户分层分析（LTV分层）
- 库存健康预警系统

**第三阶段（6-12个月）：预测与自动化**
- 销售预测模型
- 智能补货推荐
- 自动化报告推送

## 二、对不对的判断

**✅ 核心框架正确：**
- SP API覆盖模块的描述与亚马逊官方文档一致
- 独立站核心指标体系（流量-转化-订单-客户-财务）是行业标准框架
- 数据整合三种模式的渐进路径符合实际业务发展规律

**⚠️ 需要补充/注意的：**
- SP API在中国大陆地区访问需要稳定的海外网络环境
- SP API有速率限制（请求频率限制），大规模数据提取需要申请提升配额
- Looker Studio免费版有数据行数限制（10万行），超大规模数据需要付费版
- 财务数据涉及收款信息，跨境电商资金流水建议定期导出备份

**📌 实践注意点：**
- SP API的Orders API和Finances API是分开授权的，权限申请时需要注意
- GA4与Shopify的集成需要配置UTM参数和Google Ads账号关联才能完整归因
- 第三方BI工具如DataHawk通常有数据延迟（通常T+1），实时性要求高需要自建
