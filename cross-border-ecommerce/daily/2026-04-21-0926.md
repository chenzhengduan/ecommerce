## 一、核心概念

### 1. IPI（Inventory Performance Index）是什么

亚马逊库存绩效指标（IPI）是衡量卖家库存管理能力的综合评分，分值范围 0~1000。

**IPI 四大考核维度：**

| 维度 | 权重 | 说明 |
|------|------|------|
| 库存充足率 (In Stock Rate) | ~25% | 有库存且可售的SKU占比，反映备货及时性 |
| 畅销品销售转化率 (Sell-through) | ~25% | 过去90天销量/平均库存，衡量库存周转效率 |
| 滞留库存率 (Stranded Inventory) | ~25% | 无在售信息的库存（因listing问题导致无法销售的库存） |
| 超标库存率 (Excess Inventory) | ~25% | 冗余库存占比（某SKU库存可供销售天数>90天） |

**IPI 分数计算逻辑：**
- 每个维度独立打分（0~250），总和即IPI（0~1000）
- 滞销库存和滞留库存对分数影响最为直接
- 季节性产品在淡季销售转化率低属正常，不会大幅拉低IPI

### 2. FBA容量限制（Capacity Limit）

亚马逊自2024年起全面推行FBA容量管理制，替代原有季度库存限制：

**容量限制类型：**
- **标准库存上限**（Standard-Size Limit）：按体积立方英寸或千克计算
- **超大商品上限**（Oversize Limit）：独立容量池
- **营运中心容量**（FC Capacity）：按ASIN或SKU级别限制

**容量管理器（Capacity Manager）：**
- 亚马逊允许卖家在库存拥挤期购买额外容量
- 费用按仓位数/周计，不同站点差异大
- 美国站：约$0.15-$0.30/立方英尺/周（高峰期可达$0.60+）

**容量限制触发机制：**
- IPI < 400：受容量限制（标准+超大均受限）
- IPI ≥ 400：不受标准容量限制，但超大商品仍可能有单独限制
- 新卖家或新入仓商品有90天免限制期

## 二、FBA库存管理核心工具

### 1. 库存仪表板（Inventory Dashboard）

路径：卖家中心 → Inventory → Manage Inventory → Inventory Dashboard

关键子模块：
- **FBA Inventory Age Report**：库存周转天数分析，存放>180天开始收长期仓储费
- **Restock Report**：建议补货SKU和数量（基于历史销量+当前库存）
- **Inventory Health Report**：滞留/超龄/滞销库存预警
- **Stranded Inventory Report**：因listing问题无法销售的库存
- **Excess Inventory Report**：冗余库存建议（可供>90天销售的库存）

### 2. 库存预言家（Inventory Planner）

亚马逊官方数据预测工具：
- 显示每个SKU未来4-8周预计库存水平
- 红色预警：预计断货日期
- 绿色状态：库存充足
- 建议补货数量 = 覆盖周期销量 + 安全库存 - 当前库存 - 在途库存

### 3. 亚马逊物流库龄报告（FBA Inventory Age）

- 分仓龄段统计：0-90天 / 91-180天 / 181-270天 / 271-365天 / >365天
- 181天以上库存开始收取长期仓储费（LTSF）
- 长期仓储费：美国站 $6.90/立方英尺（>365天），英国站 £6.40/立方英尺

## 三、IPI优化实战策略

### 1. 提高库存充足率（In Stock Rate）

**目标：保持>95%的有库存可售SKU占比**

策略：
- 开启"冗余库存自动修复"功能（自动补货推荐）
- 使用Restock Report设定补货提醒（提前4-6周备货）
- 设置卖家中心库存预警（低库存阈值提醒）
- 头程空运+海运组合：爆款空运补急用，海运备大批量
- 避免断货：安全库存 = 日均销量 × 备货周期 + Buffer（1.2-1.5系数）

### 2. 提升销售转化率（Sell-through）

**Sell-through Rate = 90天总销量 / 平均库存量**

**提升方法：**
- 优化Listing（标题/图片/价格/评价）提高转化率
- 投放PPC广告提升销量
- 参加Deal（LD/BD/DOTD）快速清货
- 降价促销（配合Coupon/Prime折扣）
- 调整广告竞价和预算分配
- 季节性/节日产品提前清仓

**不同Sell-through对应的库存策略：**

| Sell-through | 评估 | 策略 |
|-------------|------|------|
| >6 | 优秀 | 立即补货，考虑增加库存 |
| 3-6 | 良好 | 维持当前库存水平 |
| 1-3 | 一般 | 优化Listing或降价 |
| <1 | 差 | 清仓处理，停止补货 |

### 3. 消除滞留库存（Stranded Inventory）

**滞留库存产生原因：**
- Listing被删除或停售（产品下架/类目审核）
- 产品主图/标题/属性违规
- UPC/EAN问题
- ASIN与产品不匹配

**消除方法：**
1. 修复listing问题（重新激活/编辑信息）
2. 创建移除订单（Create Removal Order）：转至海外仓或销毁
3. 申请listing重新审核
4. 为滞留库存创建新listing（慎用，需重新定价）

### 4. 减少冗余库存（Excess Inventory）

**定义：某SKU库存可供销售天数>90天**

**处理方式：**

| 方式 | 适用场景 | 成本 |
|------|---------|------|
| 亚马逊物流批量清货（Liquidations）| 冗余库存但仍有价值 | 回收约5%-20%价值 |
| 奥特莱斯限时促销（Outlet Deals）| 高质量冗余库存 | 促销费用+约10-30%折扣 |
| 降价促销+Coupon | 轻微冗余（91-180天） | 损失部分利润 |
| 创建移除订单至第三方海外仓 | 库存有价值但亚马逊费用高 | 移除费用+海外仓租 |
| 销毁（作为最后手段）| 无价值库存 | 移除费用（0.15-0.30/件） |
| 广告+促销组合清仓 | 任何冗余库存 | 广告费+折扣成本 |

**FBA批量清货计划（FBA Liquidations）：**
- 亚马逊帮你将库存卖给清货商
- 回收价值约为售价的5%-20%
- 优点：省去移除到第三方仓库的麻烦和费用
- 缺点：回收价值低，不适合高价值商品

### 5. FBA容量优化

**容量计算：**
- 标准尺寸商品：按商品体积（长×宽×高/139）计算
- 相同体积但重量不同以实际重量为准

**增加容量方法：**
1. 提高IPI分数至400以上（解除标准容量限制）
2. 使用容量管理器购买额外容量（美国站已上线）
3. 申请FBA New Product Launcher（新 品入仓90天无限制）
4. 加入SEND（Send to Amazon）计划可能有额外容量奖励
5. 超大商品单独管理：大件提前规划，避免被容量限制卡住

## 四、FBA库存管理最佳实践

### 1. 分层库存管理法

将SKU按周转速度分三层：

| 层级 | 定义 | 管理策略 |
|------|------|---------|
| A类（高速周转） | 周转<30天，销量稳定 | 高库存水位，优先补货，空运补急 |
| B类（中速周转） | 周转30-90天 | 常规库存管理，海运为主 |
| C类（低速周转） | 周转>90天 | 清仓或减少备货，考虑淘汰 |

### 2. 安全库存公式

**安全库存 = Z值 × σ(需求标准差) × √(补货提前期)**

简化版：
```
安全库存 = 日均销量 × (最大备货周期 - 平均备货周期) × 1.5系数
```

**补货点 = 日均销量 × 备货周期 + 安全库存**
**补货量 = 日均销量 × 目标库存天数 - 当前库存 - 在途库存**

### 3. 季节性产品管理

- 旺季前2-3月开始集中补货
- 旺季结束后立即清仓（避免长期仓储费）
- 季节性产品单独库存池管理，不与常规品混淆
- 预估销量用历史同比数据×市场增长率

### 4. 多站点库存分配

- 避免所有库存集中在单一运营中心（减少分仓等待时间）
- 使用亚马逊库存配置服务（Inventory Placement Service）：合仓费约$0.30-$0.40/件，但提升配送速度
- 跨站点调拨（Cross-border Inventory）：欧洲站五国库存共享（EFN）可降低成本

## 六、常见误区

1. **断货才补货**：应提前4-6周监控，断货期间BSR下降影响权重
2. **IPI越低容量越少**：容量限制只看IPI<400，>400不一定有无限容量
3. **冗余库存等自然清完**：越等长期仓储费越高，应主动清仓
4. **忽视滞留库存**：滞留库存不仅影响IPI，还会被收取月度仓储费
5. **只关注销量不看库龄**：大量短期热销≠库存管理好，长期库龄会拉低IPI
