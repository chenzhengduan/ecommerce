## 一、核心概念：亚马逊A9算法与SEO逻辑

亚马逊的A9（已逐步演进为COSMO）算法核心目标是**转化率最大化**。与Google SEO不同，亚马逊SEO不看重外链和域名权重，看重的是：

- **关键词相关性**（文本匹配度）
- **历史转化数据**（点击率、转化率、销售额）
- **库存与可用性**
- **价格竞争力**
- **买家满意度**（评分、评价数量）

### Search Terms在哪里？

Search Terms是后台listing优化中的一个字段（隐藏字段），买家看不到但算法会抓取：
- 后台位置：库存 → 管理库存 → 编辑 → Keywords
- 字符限制：美国站最多249字节（约249个英文字符），欧洲站5个字段各各249字节
- 日本站：5个Search Term字段

## 二、关键词分类体系

### 按用户意图分类

| 类型 | 说明 | 示例 |
|------|------|------|
| **宽泛词（Head Term）** | 大流量、高竞争 | "running shoes" |
| **核心词（Core Term）** | 中等流量，明确产品类型 | "men running shoes" |
| **精准长尾词（Long Tail）** | 低流量高转化，精准需求 | "men mesh breathable running shoes" |
| **场景词（Use Case）** | 使用场景描述 | "gym workout shoes" |
| **人群词（Demographic）** | 目标用户描述 | "women trails running shoes" |
| **属性词（Attribute）** | 材质/颜色/尺寸/功能 | "leather waterproof hiking boots" |

### 按流量阶段分类（漏斗模型）

- **ToFu（顶部）：** "shoes for men" → 需求宽泛
- **MoFu（中部）：** "best running shoes for flat feet" → 开始对比
- **BoFu（底部）：** "Nike Air Zoom Pegasus size 10 blue" → 购买意图明确

## 三、Search Terms最佳实践

### ✅ 正确做法

1. **词根拆分，避免重复**
   - 好："`running shoes men breathable lightweight`"
   - 坏："`running shoes, shoes running, men running shoes`"（重复浪费字节）

2. **大小写不敏感** → 全部用小写

3. **单复数不用重复写** → 算法自动匹配

4. **善用同义词和英美拼写差异**
   - UK: "colour" vs US: "color"
   - UK: "organisation" vs US: "organization"

5. **属性词打散排列**，覆盖更多搜索场景

6. **竞争对手品牌词（谨慎使用）**
   - 可以写但不要诋毁，否则有法律风险
   - 如："`compatible with apple airpods case`"

7. **容量不足时的优先级**
   1. 核心精准长尾词
   2. 变体/属性词
   3. 同义词/英美差异
   4. 场景词

### ❌ 错误做法

- 放入竞品品牌名（可能触发侵权）
- 堆砌重复词（浪费容量）
- 放入ASIN或URL
- 放入敏感词/违禁词（Amazon禁止）
- 超过字符限制（超出会被截断）

### 关键词筛选标准

- 月搜索量 > 500（太低没流量）
- 竞争度适中（首页垄断程度）
- 与产品高度相关（低相关性词拉低转化率）
- 有商业购买意图（不是信息搜索词）

## 五、Listing各位置关键词布局

### 标题（Title）— 最权重位置

- 顺序公式：`[品牌] + [核心关键词] + [功能/属性] + [规格/数量] + [场景/人群]`
- 标题权重从前到后递减
- 核心关键词放最前面
- 包含适量关键词但不能堆砌（ readability 重要）
- 字符数：英文约200字符以内

### 五点描述（Bullet Points）

- 每个bullet包含1-2个关键词
- 自然融入，不要硬塞
- 突出卖点和解决问题

### 后台Search Terms（隐藏字段）

- 集中放标题和bullet放不下的补充词
- 顺序不敏感（算法会索引全部）
- 用空格分隔，不要用逗号

### 描述/Description

- A+页面内容中自然嵌入关键词
- 对SEO权重较低，但对转化重要

### 类别节点（Browse Node）

- 选择最精准的类目节点
- 选错节点=流量损失

## 六、高级策略

### 1. 关键词金字塔模型

```
顶层（头部）：高流量核心词 → 标题最前面
中层（腰部）：属性/场景词 → Bullet Points
底层（长尾）：大量长尾词 → Search Terms
```

### 2. 竞品ASIN关键词反查策略

- 找到BSR前3竞品ASIN
- 用Cerebro反查其核心关键词
- 筛选出自己产品还没覆盖的词
- 补充进Search Terms

### 3. 季节性关键词管理

- Q4节日词提前布局（Christmas, gift, holiday）
- 季节性词单独listing/变体管理
- Search Terms季节词动态更新

### 4. 否定关键词（Negative Keywords）

- 广告中否定不相关搜索词
- 避免浪费广告费，提升ACOS
- Search Terms本身不设否定（自动过滤低相关）

## 七、常见误区

| 误区 | 真相 |
|------|------|
| "Search Terms越多越好" | 容量有限，精简精准 > 堆砌 |
| "标题和ST写相同词" | ST是补充，不是复制 |
| "排名上去就完事了" | 转化率才是终极指标，排名会被高竞争挤下去 |
| "忽视长尾词" | 长尾词竞争小、转化高，是新品突破口 |
| "Search Terms可以放品牌名" | 可能侵权，除非是"兼容xxx"类描述 |

## 八、COSMO算法演进（2024+）

Amazon正在从A9向COSMO算法过渡，核心变化：

- **语义搜索**：不再只匹配关键词字面，而是理解用户意图
- **个性化推荐**：基于用户行为千人千面
- **对话式搜索**：Alexa语音搜索影响关键词策略
- **对SEO的影响**：自然语言/问题式关键词更重要（Who/What/How/Where）

**应对策略：**
- 标题和描述更多使用自然语言
- FAQ/A+内容增加问题式关键词覆盖
- 长尾词中增加"how to"、"best for"类词

## 学习内容摘要

亚马逊SEO关键词体系以**A9/COSMO算法**为核心，权重分配为：标题 > 五点描述 > Search Terms > 描述。关键词分宽泛词、核心词、长尾词、场景词、人群词、属性词六类。Search Terms字段限制249字节，要避免重复、善用同义词和英美拼写差异、用工具（Helium10 Cerebro/Jungle Scout）反查竞品关键词。listing各位置关键词布局要遵循"精准在前、属性打散"原则。COSMO算法强调语义理解和自然语言处理，对SEO策略提出新要求。

## 对不对的判断

✅ 主体内容正确，Search Terms字符限制细节以亚马逊最新政策为准（2025年后欧洲站统一为249字节格式）。❗ COSMO算法2024年后逐步上线，部分功能仍在灰度测试。❗ 否定关键词仅适用于广告，不适用于Search Terms字段。

部分新知：COSMO算法演进部分、关键词金字塔实操模型、Search Terms容量管理细节。对已有基础的人是复习+强化；对新手是系统化框架建立。
