## 一、变体基础知识

### 1.1 什么是变体（Variation）
变体是亚马逊上同一个ASIN下不同属性（如颜色、尺寸、材质、数量等）的商品组合。变体分为：
- **父项（Parent）**：不单独销售，是子商品的聚合页面
- **子项（Child）**：实际可购买的SKU，有独立库存

### 1.2 常见变体类型
| 类型 | 示例 | 适用场景 |
|------|------|---------|
| 颜色变体 | T恤：红/蓝/黑 | 服装、家居 |
| 尺寸变体 | 鞋子：38/39/40码 | 鞋类、服饰 |
| 数量变体 | 桌布：2件/4件/6件 | 家居日用 |
| 组合变体 | 套餐A/B/C | 促销套装 |
| 颜色+尺寸矩阵 | T恤：3色×4码=12个SKU | 服装大类 |

### 1.3 变体关系图
```
Parent ASIN（不可售）
├── Child ASIN 1（颜色：红，尺寸：S）
├── Child ASIN 2（颜色：红，尺寸：M）
├── Child ASIN 3（颜色：蓝，尺寸：S）
└── Child ASIN 4（颜色：蓝，尺寸：M）
```

## 二、变体创建方法

### 2.1 后台手动创建
1. 库存 → 添加产品 → 进入"变体"选项
2. 选择变体主题（Variation Theme）：如"颜色"或"尺寸"或"颜色+尺寸"
3. 填入父商品信息
4. 为每个子商品填入具体属性

### 2.2 表格上传（Inventory Loader）
适合批量创建或复杂变体关系：
```
product_id_type: ASIN
item_id: [父ASIN]
parent_child: [parent|child]
parent_product_id: [父ASIN]
relationship_type: [variation]
variation_theme: [Size-Color]
size_name/color_name: [具体值]
```

### 2.3 变体主题类型
- **单维度变体**：只有颜色或只有尺寸
- **双维度变体**：颜色+尺寸（矩阵变体）

## 三、变体策略核心要点

### 3.1 变体合并（Merging）
将多个独立ASIN合并为一个变体，实现**评论聚合**：
- 合并前：3个ASIN，各10条评论，转化率低
- 合并后：1个变体，30条聚合评论，搜索权重提升

**合并条件：**
- 商品必须相同（相同产品，仅属性不同）
- 类目支持变体
- 不是敏感类目（母婴、食品、医药）

### 3.2 变体拆分（Splitting）
将一个变体拆分为多个独立ASIN：
- 某个子商品差评过多，影响整体
- 单独推广某个颜色/尺寸
- 准备清仓某个子商品

**拆分步骤：**
1. 在后台找到该变体
2. 编辑 → 删除某个子项
3. 子项成为独立ASIN

### 3.3 变体拆分 vs 合并对比

| 操作 | 目的 | 风险 | 适用场景 |
|------|------|------|---------|
| 合并 | 聚合评论，提升权重 | 差评聚合/类目审查 | 新品冷启动/评论少 |
| 拆分 | 单独运营/隔离差评 | 丢失变体权重 | 差评集中/清仓 |
| 添加子项 | 扩充产品线 | 需重新推排名 | 新颜色/新尺寸上市 |

## 四、评论聚合策略（核心价值）

### 4.1 为什么评论聚合这么重要
- 亚马逊搜索排名算法考虑**历史累计评论**
- 聚合后变体页面总评论数=所有子项之和
- 买家看到评论多→信任度提升→转化率提高

### 4.2 聚合实操SOP
```
Step 1: 准备多个同款不同色的链接（各10-20条评论）
Step 2: 确认类目允许变体
Step 3: 使用表格模板建立父-子关系
Step 4: 上传后等待2-4周稳定
Step 5: 开始推广主推子项
```

### 4.3 聚合后的推广策略
- **主推一款**：其他子项作为陪衬
- **评论共享**：新品分享老品评论
- **BSR连带**：一个子项爆款带动其他子项

## 五、测款策略

### 5.1 变体测款流程
```
新品上架 → 多色/多款同时上架变体 → 观察7-14天数据 →
哪个子项表现好 → 单独拆分重点推广 → 其他子项维持/清仓
```

### 5.2 数据指标
| 指标 | 含义 | 决策阈值 |
|------|------|---------|
| 转化率 | 每个子项的转化 | 低于整体均值50%考虑下架 |
| ACOS | 广告效率 | 高于均值30%考虑优化 |
| 库存周转 | 哪个颜色卖更快 | 滞销色拆分清仓 |
| 搜索排名 | 关键词排名 | 主词排名持续下滑考虑策略调整 |

### 5.3 测款失败处理
- **拆分差评子项**：独立后清仓
- **添加新变体**：用新颜色/款式替换
- **完全拆分**：放弃变体，单独运营

## 六、变体合并风险与避坑

### 6.1 高风险行为
1. **跨类目合并**：不同产品强行合并 → ASIN被删除
2. **敏感类目合并**：母婴/食品/医药 → 触发类目审查
3. **虚假变体**：颜色实际相同只是名称不同 → 违规
4. **频繁拆分合并**：操纵评论嫌疑 → 账号风险

### 6.2 合规红线
- 商品必须真实是同一产品（仅属性不同）
- 不能通过变体操纵排名
- 变体主题必须与类目匹配
- 透明计划商品可合并但需一致

### 6.3 关联销售（FBT - Frequently Bought Together）
- 变体自动展示在详情页
- 可人为创建FBT推荐（配套销售）
- 与变体不同：FBT是不同ASIN的捆绑

## 七、进阶玩法

### 7.1 变体+Bundle捆绑
- 变体子项+互补品打包
- 提升客单价
- 增加搜索词

### 7.2 变体+ Deal
- 变体中某一子项单独报Deal
- 其他子项正常销售
- 注意Deal与Coupon叠加规则

### 7.3 跨站点变体策略
- 美国站爆款 → 欧洲站建立变体
- 评论同步（部分类目）
- 降低新站点冷启动成本

## 学习摘要

亚马逊变体策略是精细化运营的核心技能之一，通过**合并/拆分/添加子项**三种操作，配合**评论聚合、测款、关联销售**等策略，实现流量和转化的最大化。核心价值在于：
1. 评论聚合提升搜索权重
2. 测款降低新品风险
3. 关联销售提升客单价
4. 灵活拆分隔离差评影响

合规红线：必须是真实同一产品，不能通过变体操纵排名或评论。
