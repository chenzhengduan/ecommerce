## 一、亚马逊Deal类型全解析

### 1. Lightning Deals（LD，秒杀）
- **定义**：限时限量的促销活动，在Deal页面有倒计时显示
- **费用**：美国站 $150~$1,000/次（根据类目和时段）；欧洲站 €35~€500；日本站 ¥1,000~¥10,000
- **时长**：通常4~12小时，欧洲站可达24小时
- **最低评分要求**：通常3.5星以上（具体类目有差异）
- **最低库存要求**：通常需要准备活动期间预计销量的库存（一般为平时周销量的2~4倍）
- **最佳效果条件**：评分4.0+，折扣足够（20%+），库存充足，评分高于竞品

### 2. Best Deals（BD，七天促销）
- **定义**：持续7天的促销活动，出现在Today's Deals页面
- **费用**：通常免费或$150（美国站）
- **时长**：7天连续展示
- **最低要求**：通常无评分要求（新品可参与）
- **优势**：展示时间长，转化率比LD稳定，没有倒计时压力
- **适用场景**：新品推广、库存清理、节假日促销

### 3. Deal of the Day（DOTD，镇店之宝）
- **定义**：每天只有3~4个产品的黄金位置促销，持续24小时
- **费用**：通常免费（但需要亚马逊邀请）
- **要求**：通常需要年销售额$1M+，品牌备案，评分4.2+；需通过客户经理或招商经理申请
- **转化率**：极高（通常为平日销售的5~10倍）
- **注意**：DOTD与LD/BD通常不可同时申报，需错开时间

## 二、Coupon（优惠券）策略

### 设置条件
- **最低折扣**：美国站要求至少5% OFF
- **设置费用**：$0.60/张（每兑换一次，无论折扣大小）
- **预算控制**：可设置总预算（如$500），用完即止
- **人群定向**：可设置仅Prime用户可用（美国站）

### Coupon类型
| 类型 | 说明 | 适合场景 |
|------|------|---------|
| 金额Coupon | 如$3 OFF | 客单价>$15的产品 |
| 百分比Coupon | 如10% OFF | 客单价<$30或清仓产品 |
| Prime专享Coupon | 仅Prime用户可见 | 想冲Prime Day排名的产品 |

### Coupon效果数据参考
- 设置Coupon的产品，转化率通常提升**5%~15%**
- Prime专享Coupon效果更好（转化率额外提升**3%~8%**）
- Coupon+Deal叠加效果最佳（见下文）

## 三、Deal与Coupon叠加玩法（核心重点）

### 叠加逻辑（美国站）
**Deal折扣 ≠ 最终到手价**，Coupon可在Deal价基础上再叠加：

```
示例（美国站）：
产品标价：$50
LD折扣：20% OFF → Deal价：$40
Coupon：10% OFF → 折上折：$40 × 0.90 = $36
```

**欧洲站注意**：欧洲站促销页面没有"Promotion"标签，不显示叠加结果，消费者看到的是最终价（已自动计算）。

### 叠加公式
| 促销类型 | 可叠加Coupon？ | 说明 |
|---------|-------------|------|
| LD（Lightning Deal） | ✅ 可以 | 可在LD价基础上叠加Coupon |
| BD（Best Deal） | ✅ 可以 | 可在BD价基础上叠加Coupon |
| DOTD（Deal of the Day） | ⚠️ 通常不可 | DOTD通常与Coupon互斥，需与AM确认 |
| Prime专享折扣 | ❌ 不可 | Prime专享折扣与Coupon不可叠加 |

### 叠加效果案例
**案例：某$50耳机叠加策略**
```
第一步：LD 20% OFF → $40
第二步：Coupon 10% OFF → $36
第三步：Prime专享（若同时开启）→ 与Coupon互斥，选其中更优惠的

实际效果：$36 ≈ 72折
如果没有Coupon：$40 ≈ 80折
差异：看似不大，但转化率差异显著
```

## 四、Prime会员专属折扣（Prime Exclusive Discount）

### 设置条件
- 必须为Prime会员设置专属价格
- 折扣幅度：至少5% OFF
- 可设置最高立减金额（$、€等）
- 需品牌已进行亚马逊品牌备案（Brand Registry）

| Coupon | ❌ 不可叠加，系统报错 |
| 促销活动（Promotion） | ✅ 可以叠加（独立折扣码）|

## 五、最佳Deal叠加组合策略

### 组合1：LD + Coupon（最常见）
```
策略：提报LD的同时，设置一张Coupon（建议10-15% OFF）
效果：双重优惠标签（"Up to X% OFF" + "Clip Coupon"），视觉冲击更强
适合：想快速冲量、提升BSR排名的产品
```
**风险控制**：设置Coupon预算上限，避免超支。

### 组合2：BD + Coupon（更适合长线推广）
```
策略：BD持续7天，同时设置一张Coupon持续7天
效果：稳定的转化率提升，没有LD的时效压力
适合：新品推广、旺季预热
```

### 组合3：LD + Prime专享折扣 + 社交媒体促销码
```
策略：LD期间，同时开启Prime专享折扣和社交媒体促销码（独立码）
注意：Prime专享折扣与Coupon不可叠加，但与独立促销码可叠加
效果：最大化价格吸引力
适合：站外引流到亚马逊购买
```

### 组合4：DOTD + 站外引流（最爆量组合）
```
策略：申请DOTD → 通过Facebook/Instagram/TikTok发布Deal链接
      → 站外流量涌入 → 站内BSR快速上升 → 反哺自然排名
风险：DOTD断货或物流跟不上会导致排名回落
```

## 七、Deal避坑指南（重要）

### ❌ 常见错误
1. **库存准备不足**：Deal中途库存断货，亚马逊自动取消Deal，权重受损
2. **折扣叠加算错价**：设置Coupon前没有计算好叠加后价格，导致亏损
3. **评分不够跑Deal**：3星以下跑Deal，大量差评涌入，BSR反而下降
4. **DOTD与Coupon同时开启**：系统报错，促销失败
5. **频繁跑同一ASIN的Deal**：亚马逊会识别为"Deal依赖"，自然排名下降
6. **欧洲站叠加标签不显示**：欧洲站促销无Promotion标签，不要误以为没有叠加

### ✅ 正确做法
- **提前计算叠加价格**：用Excel/Python算好所有折扣组合的到手价
- **提前检查评分**：Deal前确保评分4.0+，必要时用Vine计划提升评分
- **准备充足库存**：Deal库存 = 预计销售量 × 1.3（保险系数）
- **设置Coupon预算上限**：避免超出预算
- **错开DOTD和Coupon**：DOTD前24小时关闭Coupon，Deal结束后再开启

## 八、Coupon与Deal组合ROI计算模板

```
已知：
- 产品标价：$50
- 产品成本：$18
- 头程+FBA费用：$12
- LD折扣：20% OFF
- Coupon：10% OFF

计算：
LD价 = $50 × 0.80 = $40
Coupon叠加后 = $40 × 0.90 = $36

单笔利润 = $36 - $18（成本） - $12（FBA） = $6
利润率 = $6 / $36 = 16.7%

对比不做促销：
标价利润 = $50 - $18 - $12 = $20（利润率40%）

结论：Deal+Coupon叠加后利润率从40%降到16.7%，
但若能带来足够的BSR提升和评论增长，长线ROI可能更优
```

## 学习内容摘要

亚马逊Deal叠加玩法是站内促销的核心组合策略：
- **三类Deal**：LD（4-12小时，$150-$1000）、BD（7天，免费/低费）、DOTD（24小时，邀请制）
- **Coupon叠加**：在Deal价基础上可再叠加Coupon（10%+ OFF，$0.60/张），美国站可见叠加标签，欧洲站无标签但系统自动计算
- **Prime专享折扣**：与Coupon互斥，但可与独立促销码叠加
- **最佳组合**：LD+Coupon（最常见）、BD+Coupon（长线稳定）、DOTD+站外引流（爆量）
- **避坑核心**：库存充足、评分够用、折扣算清楚、DOTD和Coupon不同开
- **申报节奏**：Prime Day提前8周开始准备，Black Friday提前10周
