## 一、库存管理的核心地位

亚马逊运营中，**库存管理是生死线**。断货直接导致BSR排名下滑、关键词权重稀释；库存积压则带来长期仓储费吞噬利润甚至亏损。库存管理的本质是**在"不断货"和"不积压"之间找到平衡点**。

---

## 二、亚马逊库存关键指标

### 2.1 库存绩效指标（IPI）

亚马逊IPI（Inventory Performance Index）分数满分1000，分数<450会影响FBA仓储限制。

**IPI四大维度：**

| 维度 | 权重 | 优化方向 |
|------|------|---------|
| 过剩库存率 | 25% | 减少冗余库存，加快周转 |
| 滞销库存率 | 25% | 清理180天+无销售库存 |
| 有现货率 | 25% | 保持热卖商品不断货 |
| 库存周转率 | 25% | 提升销售速度匹配补货节奏 |

**IPI参考分数：**
- 450分以下：受限（受影响品类仓储容量）
- 450-700分：标准容量
- 700分以上：获得更多FBA容量权限
- 850分以上：解锁FBA容量管理器优先权

### 2.2 关键库存指标

| 指标 | 计算公式 | 健康值 |
|------|---------|-------|
| 库存周转天数 | 平均库存 ÷ 日均销量 | 30-60天 |
| 现货率 | 有货天数 ÷ 总天数 | ≥ 95% |
| 滞销率 | 90天未售库存 ÷ 总库存 | < 10% |
| 过剩率 | 180天未售库存 ÷ 总库存 | < 5% |
| 安全库存天数 | 断货风险敞口的天数 | 7-14天 |

## 三、断货防控体系

### 3.1 断货的危害

断货对亚马逊Listing的伤害是**长期且不可逆的**：

1. **BSR排名下滑**：断货超过7天，排名可能从Top 100跌至Top 1000+
2. **关键词权重稀释**：断货期间广告权重积累暂停，恢复后需重新积累
3. **竞品抢占份额**：断货期间竞品趁机获取原本属于你的流量
4. **Listing活跃度下降**：亚马逊算法降低对不活跃Listing的推送权重

### 3.2 断货预警机制

**三级预警体系：**

```
红色预警（立即补货）
  └─ 库存可售天数 < 安全库存天数
  └─ 当前库存 < 7天平均销量

橙色预警（密切监控）
  └─ 库存可售天数 = 安全库存天数 × 1.5
  └─ 当前库存 < 14天平均销量

黄色预警（计划补货）
  └─ 库存可售天数 = 安全库存天数 × 2
  └─ 当前库存 < 21天平均销量
```

**预警监控工具：**
- 亚马逊库存管理后台（手动定期检查）
- 第三方工具：Helium 10、Inventory Shield、SoStocked
- 自建预警系统（通过SP API对接）

### 3.3 补货公式详解

**基础补货公式：**

```
补货量 =（备货天数 + 安全库存天数）× 日均销量 - 当前在途库存
```

其中：
- **备货天数**：从下单到FBA入仓的时长（头程时效）
- **安全库存天数**：防止意外断货的缓冲（通常7-14天）
- **日均销量**：过去7-30天平均值（建议取加权平均，近期权重更高）
- **当前在途库存**：海运/空运中尚未到达FBA的库存

**加权平均日均销量计算：**
```
日均销量 =（近7天销量 × 3 + 近14天销量 × 2 + 近30天销量 × 1）÷ 6
```

**实际案例：**
> 产品A：  
> - 近7天销量：210件（30件/天）  
> - 近14天销量：350件（25件/天）  
> - 近30天销量：750件（25件/天）  
> - 加权日均销量 =（210×3 + 350×2 + 750×1）÷ 6 = 1225 ÷ 6 ≈ 204件/天  
> - 头程时效：15天，安全库存：10天  
> - 当前库存：500件，在途：1000件  
> - 补货量 =（15+10）× 204 - 500 = 5100 - 500 = 4600件

### 3.4 补货决策树

```
评估维度：
    ├─ 日均销量稳定性（波动大→多备安全库存）
    ├─ 头程时效波动性（旺季时效不稳→提前补货）
    ├─ 资金压力（资金紧张→小批量多批次）
    └─ 仓储费压力（长期仓储费高→控制批量大小）

决策输出：
    ├─ 即时补货（断货风险高）
    ├─ 计划补货（正常周期）
    └─ 暂缓补货（库存过剩）
```

## 四、库存过剩防控

### 4.1 库存过剩的危害

| 后果 | 说明 |
|------|------|
| 长期仓储费 | 亚马逊费率：1月-9月 $6.90/立方英尺；10月-12月 $22.45 |
| 现金流占用 | 资金被库存套牢，无法周转 |
| 库存老化 | 超过180天滞销，亚马逊可能自动移除 |
| IPI降分 | 滞销库存拉低库存绩效分数 |
| 亡命货 | 长期卖不动的库存，最终可能只能销毁 |

### 4.2 库存过剩信号识别

**早期信号：**
- 连续4周销量下滑超过20%
- 竞品出现大幅降价抢量
- 类目整体需求季节性下降
- 评论星级下降（质量投诉增加）

**量化判断标准：**
```
库存可售月数 = 当前库存 ÷ 月均销量
> 3个月：警告（考虑清仓）
> 6个月：危险（必须行动）
> 12个月：紧急（长期仓储费将大幅增加）
```

### 4.3 库存过剩处理策略

**策略一：主动降价促销**
- 阶段性地降低价格（每次降5-10%，观察销量变化）
- 使用亚马逊Coupon（5%-30%折扣）
- 报LD/BD/DOTD等Deal活动快速清货
- 优势：快速出量；缺点：利润率下降甚至亏损

**策略二：移除冗余库存**
- 创建移除订单（Create Removal Order）：亚马逊代为处理
- 可选择：退回到指定地址、销毁、捐赠（Scouts）
- 适合：高价值但周转极慢的商品

**策略三：FBA Liquidations（清算）**
- 亚马逊帮你把库存卖给清算人
- 回收价格通常是原价的5%-20%
- 操作简单，但回收价值低
- 适合：客单价低、周转极慢的冗余库存

**策略四：移仓到第三方海外仓**
- 移出FBA，避免长期仓储费
- 在海外仓重新贴标，重新入仓或通过独立站等其他渠道销售
- 适合：高价值、有其他销售渠道的商品

**策略五：捆绑销售（Bundle）**
- 将滞销品与热销品捆绑销售
- 通过变体合并或创建Bundle套装
- 适合：有配件关联性或人群关联性的产品

**策略六：站外Deal引流清仓**
- 在Slickdeals/Dealabs等折扣平台发布Deal
- 站外放量快速提升排名，配合站内广告拉回
- 适合：有一定评价基础的商品

## 五、季节性与波动性管理

### 5.1 季节性库存规划

季节性产品（如万圣节、圣诞节、夏季用品）需要提前3-6个月规划：

```
季节性产品补货时间线：
  T-6个月：确定季节需求预测，计算备货量
  T-4个月：下单生产/采购
  T-2个月：海运发货（预留15-30天入仓）
  T-1个月：FBA到货，开始预热推广
  T-0（季节开始）：全力销售
  T+季节结束：立即启动清仓，避免长期仓储费
```

### 5.2 促销活动期间库存策略

**Prime Day / Black Friday / Cyber Monday期间：**
- 提前备足2-3倍日常销量库存
- 活动前30天控制广告预算，避免活动前大量消耗
- 活动后及时补货（活动后7-14天通常有销售余波）

## 六、库存管理系统搭建

### 6.1 手动管理（SOP）

适合SKU少于50个的小型卖家：

```
每日：检查红色预警SKU
每周：核查橙色预警SKU，更新补货计划
每月：审查库存绩效，清理滞销品
每季：评估季节性备货，预测下一季度需求
```

### 6.2 第三方工具管理

| 工具 | 核心功能 |
|------|---------|
| **SoStocked** | 智能补货计算、多仓库管理、自动化预警 |
| **Inventory Manager** | 库存追踪、自动补货建议、滞销提醒 |
| **Helium 10** | 库存管理模块、补货计算器 |
| **SellerMobile** | 实时库存监控、自动补货建议、报告 |
| **Forecastly** | 需求预测、补货优化 |
| **SkuGrid** | FBA库存管理、移仓管理 |

### 6.3 SP API自建系统

通过亚马逊SP API搭建自定义库存管理系统：

```python
# 核心API调用逻辑
GET /fba/inventory/ summaries # 获取实时库存数据
GET /fba/inventory/aged_data   # 库龄数据（识别滞销）
GET /fba/inventory/inbound_workship_items  # 在途库存
```

**自建系统优势：**
- 完全自定义逻辑
- 与内部ERP/WMS系统集成
- 支持多账号统一管理

## 七、库存与现金流管理

### 7.1 库存资金占用分析

```
库存投资回报率（ROI）=（销售收入 - 成本 - 费用）÷ 库存成本 × 100%

优化方向：
  ├─ 缩短补货周期（更频繁、小批量补货）
  ├─ 提高预测准确性（减少过量备货）
  ├─ 优化产品组合（高周转品占比提升）
  └─ 使用第三方海外仓（降低FBA长期仓储费）
```

### 7.2 库存在资产负债表的体现

| 项目 | 说明 |
|------|------|
| 库存成本 | 采购成本 + 头程运费 + 关税 |
| FBA费用 | 入库处理费 + 仓储费 + 配送费 |
| 潜在风险 | 滞销/过季/损坏导致的库存减值 |
