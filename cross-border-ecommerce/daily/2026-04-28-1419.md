## 一、竞品监控体系

### 1. 核心监控维度

| 维度 | 监控内容 | 工具/方法 |
|------|---------|----------|
| 价格监控 | 竞品价格变动、优惠券使用情况 | Keepa、Helium 10、Jungle Scout |
| 库存监控 | 库存数量、预计补货时间、FBA可用性 | SellerApp、Inventory Manager |
| 评分/评论 | 星级变化、新增评论数量和内容 | ReviewTrader、FeedbackExpress |
| BSR排名 | 类目排名变化、关键词排名波动 | ZonGuru、Data Boxer |
| Listing变更 | 标题、图片、A+内容、主图变化 | manual check、AMZ Tracker |

### 2. 监控频率策略

- **高频监控（每日）：** 价格、库存可用性、BSR前10名
- **中频监控（每周）：** 评论增长、评分变化、竞品新品
- **低频监控（每月）：** 竞品整体战略调整、品牌布局

### 3. 数据采集架构

```
竞品ASIN列表 → 自动爬取(API/爬虫) → 数据存储 → 分析预警 → Dashboard展示
```

关键指标阈值：
- 价格变动 > 5% → 触发预警
- 库存归零 → 触发机会提醒
- BSR排名上升 > 50名/天 → 触发关注

## 二、动态定价策略

### 1. 动态定价理论基础

**动态定价 = 根据市场需求、竞争格局、库存状况实时调整价格**

- **目的：** 最大化利润 or 最大化销量 or 市场份额优先
- **前提：** 有足够的自动调价工具和规则配置

### 2. 定价策略类型

| 策略 | 适用场景 | 风险 |
|------|---------|------|
| 竞争性定价 | 新品冷启动、市场份额争夺 | 价格战、利润压缩 |
| 价值定价 | 有差异化优势、品牌产品 | 需要足够的用户认知 |
| 阶梯定价 | 多SKU产品线、捆绑销售 | 复杂、计算难度高 |
| 时段定价 | 旺季/大促期间 | 频繁调价影响排名 |

### 3. Repricer工具

常用工具：
- **Appeagle：** 基��Buy Box价格的自动调价
- **BQool：** 实时竞品价格监控 + 自动调价
- **SellerDynamics：** 复杂规则引擎，支持多平台
- **Intelunch：** 专注于亚马逊的智能定价

### 4. 定价规则设计

**基本原则：**
1. 设定价格上下限（保护利润）
2. 追踪Buy Box持有者价格（非最低价）
3. 库存低时提价，库存高时降价
4. 大促期间提前预热价格

**参考公式：**
```
动态价格 = 基础价 ± (竞品价差 × 调整系数) ± (库存调整因子)
```

## 三、竞品监控 + 动态定价联动

### 场景联动

| 场景 | 竞品反应 | 定价应对 |
|------|---------|---------|
| 竞品降价 | 监控到价格下跌 | 判断是否跟随，考虑价值定位 |
| 竞品缺货 | 库存归零/不可售 | 趁机提价，收割流量 |
| 竞品断货恢复 | 重新补货 | 快速降价夺回市场份额 |
| 竞品获得好评 | 评分超越 | 加强自身评价管理，考虑降价 |
| 竞品做Deal | 促销期间 | 匹配Deal或差异化应对 |

### 预警系统设计

```
竞品价格下降10% + 我品库存充足 → 自动降价5%提醒
竞品缺货 + 我品BSR在该类目TOP20 → 自动提价15%提醒
竞品获得50+新评论/周 → 触发分析：是否新品来袭？
```

## 四、数据化运营要点

1. **建立竞品数据库：** ASIN、关键指标历史数据

4. **避免价格战：** 以价值而非价格作为核心竞争点
5. **关注Buy Box归属：** 了解Buy Box价格的逻辑（不仅看排名）

## 学习内容摘要

本主题涵盖两个核心体系：**竞品监控**和**动态定价**。竞品监控通过多维度数据采集（价格、库存、评论、BSR、Listing变化）建立完整的竞品情报体系；动态定价则通过工具自动化响应市场变化，在保护利润的前提下提升竞争力。两者联动形成完整的竞品攻防体系。

核心学习点：
- 竞品监控不是简单看价格，要有完整的指标体系
- 动态定价的核心是规则设计，而非越便宜越好
- 工具是手段，策略才是核心
- 避免陷入价格战泥潭
