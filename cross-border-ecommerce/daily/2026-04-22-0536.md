## 学习内容摘要

### 一、亚马逊广告归因基础概念

**什么是广告归因？**
归因是将"订单成交"功劳分配给特定广告触点的过程。亚马逊是归因的最后点击模型，但也支持多触点归因分析。

**核心指标关系：**
```
TACOS（总广告销售成本比）= 总广告 spend / 总销售额
  = (平台内广告Spend + 站外广告Spend) / (平台内销售 + 站外销售)

ROAS（广告支出回报率）= 广告带来销售额 / 广告Spend
  例如：$5 ROAS 表示每花$1广告带来$5销售额

ACOS（广告销售成本比）= 广告Spend / 广告带来销售额
  例如：20% ACOS 表示每$1销售额有$0.20用于广告
```

**关键关系：**
- ACOS越低 → 广告效率越高
- ROAS越高 → 广告效果越好（ROAS = 1/ACOS）
- TACOS比ACOS更能反映全渠道广告效率

### 二、亚马逊归因窗口

**点击归因窗口（Click Attribution Window）：**
- **1天窗口**：广告点击后1天内购买
- **7天窗口**（默认）：广告点击后7天内购买（即使中间看了其他广告也算首次点击）
- **14天窗口**：广告点击后14天内购买
- **30天窗口**：广告点击后30天内购买

**浏览归因窗口（Impression-Only Attribution Window）：**
- **1天浏览归因**：只看过广告图片但没点击，1天内购买
- **7天浏览归因**：只看过广告，7天内购买
- **浏览归因贡献**：通常占广告总订单的10%-30%

**为什么浏览归因重要？**
- 有些买家看到广告 → 不点击但记住了 → 直接搜索品牌名购买
- 这部分功劳如果只用点击归因会被遗漏
- 亚马逊报告的"浏览归因订单"帮助了解真实广告价值

### 三、亚马逊广告归因模型详解

**SISP模型（Same-IP-Same-Product）：**
- 亚马逊用于判断是否同一消费者重复购买
- 同一IP购买同一产品，只计算一次新客销售额
- 防止刷单/重复购买虚增广告效果

**最后点击模型（Last Click Attribution）：**
- 订单100%归功于最后一次点击的广告
- 亚马逊默认使用此模型计算广告报告中的订单
- 问题：忽略前期触点贡献

**多触点归因（Multi-Touch Attribution, MTA）：**
- 将订单功劳分配给购买路径上所有触点
- 例如：买家先看了SB广告（40%）→ 再看了SP广告（30%）→ 最后直接购买（30%）
- 需要Amazon Attribution标签追踪站外广告

**Amazon Attribution核心功能：**
- 为每个站外广告（Google/Facebook/邮件等）生成唯一追踪标签
- 安装在站外落地页，追踪从点击到购买的全路径
- 支持展示广告→搜索广告→购买的跨渠道归因
- 报告位置：品牌分析Brand Analytics → Amazon Attribution

### 四、TACOS vs ACOS：全渠道视角

**ACOS局限性：**
- 只计算亚马逊站内广告Spend和销售额
- 不反映站外引流对自然搜索排名的帮助
- 不反映站外直接销售（独立站等）

**TACOS全貌：**
```
TACOS = (亚马逊广告Spend + 站外广告Spend) / 总销售额

场景示例：
- 亚马逊广告Spend: $1000，亚马逊销售: $20000 → ACOS = 5%
- 站外广告Spend: $500，站外销售: $5000
- 总销售额: $25000
- TACOS = $1500 / $25000 = 6%

单看ACOS是5%很好，但TACOS是6%，说明站外广告效率相对较低
```

**何时看TACOS vs ACOS：**
- 投放亚马逊广告为主 → 看ACOS
- 有站外引流（独立站/Google/Facebook） → 看TACOS
- 品牌推广期 → 接受较高ACOS，看品牌搜索量增长

### 五、广告归因的实操应用

**1. 判断广告真实效果：**
- 开启Amazon Attribution，站外广告带来订单能在亚马逊后台看到
- 如果站外带来订单多 → 说明站外种草有效
- 如果站外带来订单少 → 说明站外内容需要优化

**2. 关键词归因分析：**
- 在广告报告中查看"归因类型"列
- 区分"点击归因订单"和"浏览归因订单"
- 核心词/品牌词通常浏览归因贡献高（买家记得品牌名）

**3. 归因窗口对数据的影响：**
- 30天窗口相比7天窗口，订单数通常多15%-25%
- 长期决策（如品牌投放）用30天窗口更准确
- 短期促销（如Prime Day）用7天窗口足够

**4. 跨渠道归因打通：**
```
路径：Facebook广告 → 点击 → 落地页（带Attribution标签）→ 加入购物车 → 离开 → Google搜索 → 购买

最后点击归因：100%给Google
多触点归因（如用Attribution）：Facebook 40% + Google 60%

解决方案：用亚马逊的"多点归因报告"或第三方工具（DataHawk/Teikametrics）
```

### 六、常见归因误区与避坑

**误区1：只看ACOS忽略TACOS**
- 站外广告可能降低ACOS但整体效率不高
- 应该综合TACOS判断全渠道ROI

**误区2：把浏览归因订单当成自然流量**
- 浏览归因本质也是广告触达，只是没点击
- 区分"纯自然订单"和"浏览归因广告订单"

**误区3：归因窗口设置过短**
- 如果只看1天窗口，会丢失7-30天内的延迟转化
- 旺季前拉长归因窗口（如Prime Day前30天开始投放）

**误区4：忽视品牌词搜索的归因价值**
- 买家看到SB广告后，搜索品牌词购买 → 品牌词广告抢走了功劳
- 解决方案：开启SB品牌投放竞品，品牌词用于截流

## 学习总结

### 内容是否正确
✅ **正确**。亚马逊归因窗口、点击vs浏览归因、SISP模型等概念与亚马逊官方文档和行业实践一致。TACOS vs ACOS分析框架是行业标准做法。

⚠️ **补充**：Amazon Attribution功能需要申请开通（Brand Registry所有者可申请），并非所有卖家默认拥有。归因数据存在一定采样偏差。

### 是否为新知识
🆕 **大部分为新知识或整合性知识**：
- 多触点归因概念之前未系统覆盖
- TACOS vs ACOS的全渠道分析框架是进阶内容
- Amazon Attribution的详细运作机制是新增内容
- SISP模型的具体应用是新增内容

### 是否好理解
✅ **好理解**。归因模型本身有逻辑层次（点击归因→浏览归因→多触点归因），与广告投放实操结合紧密，理解后能直接用于优化广告策略。概念略偏技术，但对运营人员来说属于必修知识。
