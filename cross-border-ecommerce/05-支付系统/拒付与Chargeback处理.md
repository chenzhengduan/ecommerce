# 拒付与 Chargeback 处理

> 分类：05-支付系统  
> 本页更新：2026-05-26  
> 适用范围：独立站、平台卖家、信用卡收单、PayPal 争议与跨境支付风控

## 核心判断

拒付不是单纯的客服问题，而是 **支付风控、履约证据、商品描述、物流体验和售后响应** 的综合结果。跨境卖家要把拒付管理前置到下单、发货、签收和客服流程里，而不是等银行发起 dispute 后再临时补材料。

## 拒付的基本流程

1. 买家向发卡行或支付平台提出争议。
2. 发卡行通过卡组织或支付平台发起 dispute / chargeback。
3. 支付服务商通常会先冻结或扣回交易金额，并可能收取争议费用。
4. 卖家在规定期限内提交证据。
5. 发卡行或卡组织根据原因码和证据做裁决。
6. 卖家胜诉则款项可能返还；败诉则损失交易金额、争议费和相关履约成本。

Stripe 官方文档说明，dispute 发起后会立即逆转付款，并从 Stripe 余额中扣除交易金额和争议费。PayPal Seller Protection 也覆盖部分 claims、chargebacks 和 reversals，但需要满足条件，并非所有场景都保护。

## 常见原因

| 类型 | 买家说法 | 常见根因 | 应对重点 |
|---|---|---|---|
| 未授权交易 | 我没买过 | 盗刷、家庭成员误用、买家反悔 | 3DS、设备/IP、账单地址、历史订单 |
| 未收到货 | 没收到商品 | 物流慢、无跟踪、签收异常、地址错误 | 物流轨迹、签收证明、收件地址一致性 |
| 与描述不符 | 商品不一样 | Listing 夸大、图片误导、规格不清 | 商品页截图、规格说明、客服记录 |
| 重复扣款 | 被扣多次 | 支付重试、订阅逻辑、系统错误 | 支付记录、退款记录、订单对应关系 |
| 未退款 | 商家没退钱 | 退款延迟、买家不理解到账周期 | 退款凭证、退款政策、沟通记录 |
| 友好欺诈 | 收到货后仍拒付 | 买家滥用争议机制 | 完整证据链和黑名单策略 |

## 证据链清单

### 实物商品

- 订单号、支付记录、交易时间。
- 买家姓名、邮箱、电话、账单地址和收货地址。
- 商品页面截图、规格、价格、发货时效和退款政策。
- 物流单号、承运商轨迹、派送节点、签收证明。
- 客服沟通记录、补发或退款记录。
- AVS、CVC、3D Secure、风控评分等支付验证信息。

### 虚拟商品或服务

- 账号注册信息、登录记录、IP、设备信息。
- 服务开通记录、下载记录、访问记录。
- 买家确认使用服务的邮件、工单或聊天记录。
- 订阅条款、续费提醒、取消路径和退款政策。

## 处理 SOP

### 1. 先判断是否抗辩

不要所有拒付都盲目抗辩。优先抗辩：

- 物流已签收且地址与订单一致。
- 买家已使用服务或承认收货。
- 商品页描述清晰，争议理由与事实不符。
- 存在明显友好欺诈或重复拒付行为。

建议接受或退款：

- 确实未发货或发错货。
- 物流无有效跟踪且无法证明送达。
- 商品描述存在明显误导。
- 争议金额低于处理成本。

### 2. 按原因码组织材料

证据不要堆成杂乱附件，要围绕 dispute reason 写清楚逻辑：

- 买家主张是什么。
- 事实证据是什么。
- 证据如何反驳该主张。
- 请求裁决方维持交易有效的理由。

### 3. 一次性提交完整证据

很多支付服务商和卡组织对证据提交有截止时间，且提交后不一定能反复补充。提交前应检查：

- 证据是否对应原因码。
- 物流轨迹是否完整。
- 截图是否包含 URL、时间、订单信息。
- 文字说明是否简短清晰。
- 附件是否可读、无乱码、无无关材料。

### 4. 复盘根因

每次拒付结束后，都要标记根因：

- 流量来源问题。
- 商品页描述问题。
- 履约超时问题。
- 支付验证不足。
- 客服响应慢。
- 买家欺诈或地区风险。

## 预防策略

### 支付前

- 启用 3D Secure / SCA 或动态风控。
- 对高风险国家、代理 IP、异常邮箱、账单地址不匹配订单加严审核。
- 对高客单价订单增加人工审核。
- 明确展示价格、币种、发货时效、退换货政策和订阅条款。

### 发货中

- 使用可追踪物流。
- 高客单价订单使用签收服务。
- 发货后主动发送物流信息。
- 延迟发货时提前通知买家。

### 售后中

- 保持客服响应时效。
- 对退款请求优先走主动退款，避免买家直接找银行。
- 对重复争议买家建立风险名单。
- 保存所有沟通记录。

## 指标监控

| 指标 | 含义 | 建议动作 |
|---|---|---|
| 拒付率 | 拒付笔数 / 交易笔数 | 按渠道、国家、品类拆分 |
| 欺诈率 | 被认定欺诈的交易占比 | 强化风控规则 |
| 争议胜诉率 | 胜诉拒付 / 已抗辩拒付 | 优化证据模板 |
| 退款前置率 | 主动退款 / 总争议问题 | 提前化解争议 |
| 高风险订单占比 | 被风控标记订单比例 | 调整广告流量和审核策略 |

Visa 已将欺诈和争议纳入 VAMP 等监控框架，过高的欺诈/争议水平可能触发收单行或卡组织层面的整改压力。中小卖家不一定直接对接 Visa 规则，但支付服务商会把这些风险传导到商户账户审核、保证金、冻结或关停上。

## 模板：拒付抗辩说明

```text
订单号：
交易时间：
争议原因：

事实说明：
买家于 [日期] 购买 [商品/服务]，订单金额为 [金额]。商品已于 [日期] 通过 [承运商] 发出，物流单号为 [单号]，并于 [日期] 显示签收。

证据清单：
1. 订单详情截图，显示买家信息、收货地址和商品信息。
2. 支付记录，显示交易成功。
3. 物流轨迹和签收证明。
4. 商品页面截图，显示规格、发货时效和退款政策。
5. 客服沟通记录。

结论：
该订单已按商品页承诺履约，物流信息显示送达，买家争议理由与证据不符。请求维持交易有效。
```

## 与支付工具的关系

- Stripe：重点看 dispute reason、证据要求、3DS、Radar 和争议监控。
- PayPal：重点看 Seller Protection 是否覆盖、是否发到交易详情页地址、是否能提供有效发货或送达证明。
- Adyen：适合更成熟团队做统一争议管理和本地支付风控。
- Airwallex：适合结合多币种收付和交易风险管理，但具体争议规则要看支付接受产品和账户地区。
