## Context

当前仓库已经用 MkDocs Material 承载 `cross-border-ecommerce/` 知识库，并且围绕跨境电商建立了主题地图、刷新运行手册、扫描工具和本地网页验证流程。用户希望新增一个阿里巴巴电商知识库，内容形态也应是 Markdown 文档，并且能像现有跨境电商知识库一样通过本地网页浏览。

阿里巴巴电商知识库和现有 Amazon/跨境电商知识库有交叉，但不应混在同一目录里。它需要独立主题结构，用于沉淀 1688、Alibaba.com、阿里国际站、淘系/天猫、采购批发、供应链、询盘转化、交易履约、支付结算、营销获客和风控合规等知识。

## Goals / Non-Goals

**Goals:**

- 新增独立的阿里巴巴电商知识库目录，建议使用 `alibaba-ecommerce/`。
- 建立可直接浏览的 Markdown 首页和基础目录结构。
- 将新目录接入 MkDocs，使本地网页能同时浏览跨境电商和阿里巴巴电商知识库。
- 建立阿里巴巴电商主题地图，支持后续持续补全和刷新。
- 创建首批核心笔记，覆盖平台体系、1688、Alibaba.com/阿里国际站、采购供应链、商品发布、询盘转化、履约支付、营销获客和合规风控。
- 将现有知识刷新工具泛化为可扫描不同知识库目录，而不是只支持 `cross-border-ecommerce/`。

**Non-Goals:**

- 本变更不一次性写完所有阿里巴巴电商专题。
- 本变更不迁移或重写现有 `cross-border-ecommerce/` 内容。
- 本变更不搭建新的前端应用或替换 MkDocs Material。
- 本变更不承诺自动联网持续更新；涉及时效性事实时，实施和后续刷新需要查验当前来源。

## Decisions

1. **新增 `alibaba-ecommerce/` 作为独立知识库目录。**
   - 原因：阿里巴巴体系与 Amazon/跨境电商有交叉但对象不同，独立目录能减少混乱。
   - 备选方案：放进 `cross-border-ecommerce/` 下作为一个模块。放弃原因是会让现有跨境电商目录越来越杂，且不利于未来单独刷新。

2. **调整 MkDocs `docs_dir` 为统一文档根目录。**
   - 原因：MkDocs 只能有一个 `docs_dir`。若要同时展示多个知识库，建议新增 `docs/` 或类似目录作为聚合入口，再把知识库目录纳入其下，或调整现有目录组织。
   - 备选方案：为阿里巴巴知识库单独开第二个 MkDocs 服务。放弃原因是用户希望“也要搞到 md 文档有个网页”，更自然的是沿用现有网页统一浏览。

3. **保留现有页面访问，并新增顶部/侧边入口。**
   - 原因：已有跨境电商网页正在使用，接入新知识库不能破坏旧入口。
   - 备选方案：只改 `site_name` 和根首页，不保留旧路径。放弃原因是会影响当前使用习惯。

4. **复用并泛化知识刷新机制。**
   - 原因：阿里巴巴知识库也需要长期补全和更新，现有扫描/报告机制已经可复用。
   - 备选方案：为阿里巴巴重新写一套工具。放弃原因是重复建设，后续维护成本高。

5. **首批内容采用“骨架 + 关键专题”的方式。**
   - 原因：用户要全面知识库，但一次性写满所有内容风险大。先搭结构、首页、主题地图和核心入门内容，后续按主题地图持续补齐。
   - 备选方案：首轮只建空目录。放弃原因是用户需要可浏览、可学习的知识库，而不是空壳。

## Risks / Trade-offs

- **风险：调整 MkDocs 文档根目录时破坏现有网页路径** -> 缓解方式：实施时先做最小迁移方案，并用 `mkdocs build --strict` 和本地网页访问验证旧内容仍可见。
- **风险：阿里巴巴平台规则变化快** -> 缓解方式：主题地图和笔记中标记时效性主题，涉及规则、费用、交易保障、物流、支付时查验官方来源。
- **风险：知识库范围过大** -> 缓解方式：首轮只建立核心结构和少量高优先级笔记，后续通过刷新机制逐步补全。
- **风险：跨境电商与阿里巴巴知识重复** -> 缓解方式：重复主题以“阿里巴巴场景”重写，不复制 Amazon 逻辑。
- **风险：扫描工具只支持单目录** -> 缓解方式：将工具参数化，允许指定 `--docs-dir` 和输出目录。

## Migration Plan

1. 创建阿里巴巴知识库目录和首页。
2. 建立阿里巴巴主题地图、刷新运行手册和报告模板。
3. 创建首批核心 Markdown 笔记。
4. 调整 MkDocs 聚合方式，让现有跨境电商知识库和新阿里巴巴知识库都能在本地网页访问。
5. 泛化扫描工具并为阿里巴巴目录生成首份 inventory/report。
6. 运行测试、`mkdocs build --strict`，并检查本地网页能看到两个知识库入口。

- 新 MkDocs 首页是否要改名为“电商知识库”，还是保留当前“跨境电商知识库”站点名？
